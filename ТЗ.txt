 Основные задания:

1) Сделать страницу из приложенного дизайна.
2) HTML и СSS должны быть валидными, использовать препроцессор для HTML - JADE, а для CSS - SASS.
3) Сделать мобильную и планшетную версию на свое усмотрение.
4) Использовать принцип mobile-first.
5) Для галереи использовать любой плагин или написать функционал самостоятельно.
6) Поддержка ие8+, последняя опера, сафари(мак), фаерфокс, хром.
7) Использовать google page speed insights https://developers.google.com/speed/pagespeed/insights/?hl=ru для достижения максимальных показателей по скорости загрузки страницы.
8) В процессе разработки использовать сетку, которая может быть создана вручную или взята с любого стороннего ресурса.

Дополнительные задания:

1) Использовать grunt или gulp для автоматизации работы над проектом.
2) Использовать bower для менеджмента сторонних зависимостей.
3) Данные для галереи и текстового блока http://screencast.com/t/jyf9K4p6s должны выводиться динамически, то есть при загрузке должен отправляться запрос на получение json данных, которые будут расположены в папке inc(их будет два, один для галереи, другой для блока).
При успешном запросе данные будут скомпилированы любым доступным шаблонизатором(handlebars, mustache etc.) и будут вставлены в соответствующие блоки. При загрузке в динамических блоках должен быть лоадер, после вставки он должен пропадать.

Готовый код выложить на любом ресурсе, чтобы результат был доступен по ссылке и отдельно выслать исходный код. Оценить сколько времени было потрачено на тестовое задание.
